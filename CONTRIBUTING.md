# Contributing to the OpenCompliance Site

This repo is the public explanation layer for OpenCompliance.

## Ground rules

- Keep the public/private boundary clean.
- Do not overstate maturity.
- If something is aspirational, label it aspirational.
- If a claim depends on a public repo, make sure the repo actually contains the thing being claimed.

## Good contributions

- copy edits that improve clarity,
- design and accessibility fixes,
- navigation or information architecture improvements,
- and updates that keep the site aligned with the public artifacts.

## Before opening a PR

- Keep the change set focused.
- Check that internal links still work.
- Update related docs if a page changes the public story materially.
- If a page claims a repo, artifact, or proof exists, verify that it really exists in the public foundation org.
